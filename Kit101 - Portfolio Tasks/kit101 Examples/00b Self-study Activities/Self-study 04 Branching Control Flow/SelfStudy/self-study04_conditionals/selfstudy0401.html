Self-study 4 Activity 1: Write and test conditional expressions

[ ↔ Expand↶ Close all sections ] Only visible sections will be included when printing this document.

Expected outcomes

  • Know the operators that can be used to write conditional expressions
  • Be able to use variables of type boolean
  • Be able to convert “word versions” of conditional tests into valid conditional expressions
  • Plan, and write code to implement tests of conditional expressions

Starting point

If you haven’t already, download and unzip the Self-study 4 starter files. You will be using TestCond.java in this activity.

Steps

Open the program source file TestCond.java and look at the code.

  1. List the variables of primitive types. What value is stored in each?
    Variable Identifier Type Value Stored
    number int 10
    state boolean true
    letter1 char ‘b’
    letter2 char ‘v’
  2. List the reference variables. What value is stored in each?
    Variable Identifier Contents
    word1 A reference to a String which has the contents “testing”
    word2 A reference to a String which has the contents “testing”, but note that this is a different String from word1 (This means that there will be two areas of memory set aside, both of which happen to have the same contents)
  3. Would this program compile and run as it is? If not why not? No, the empty ( ) after the if will cause a problem.

Write conditional expressions to test each of the things given in the table below. Test each of your answers in turn. Do not move on to the next condition until you are satisfied that you have the current one correct. Remember, trying something out on a few simple values is not testing! To test your expressions:

  • Write the code that you believe will implement the test into you copy of TestCond.java (the place you need to add it is marked by a comment in the code).
  • Work out values to assign to the variables so that you will be able to test whether or not your conditional expression is working as you hoped.
  • Compile and run the program as many times as it takes to be sure that you have correctly implemented the test.
Hint: Remember that if an integer (say n) is evenly divisible by another number (say k) then if we divide n by k the remainder will be zero.

The solutions below are examples; you might come up with something different that is still correct.

Note the use of the logical not operator. This operator needs just one operand and inverts its truth value. You could also write this condition as

(state == false)

but this actually results in two boolean tests being performed (the equality comparison followed by testing if that result is true and the if should be executed) so the first approach is preferred.
