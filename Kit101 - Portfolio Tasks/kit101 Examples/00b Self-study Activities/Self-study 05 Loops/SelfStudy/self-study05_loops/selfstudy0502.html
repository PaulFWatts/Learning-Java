Expected outcomes

  • Be able to solve a problem which requires the planning, implementing, and testing of a algorithm that uses a looping construct

Starting point

If you haven’t already, download and unzip the Self-study 5 starter files. You will be using NumberInfo.java and PalindromeTester.java in this activity.

Steps

1. Work out and implement an algorithm to calculate the number of digits in an integer

Open the file NumberInfo.java. This program (as currently implemented) asks the user to enter an integer and then echoes that integer to the screen. You are to add some improved functionality. You may assume that the user will always enter a positive integer.

Compile and run the program to assure yourself that it behaves as stated.

Plan and implement code to calculate how many digits the integer has and display this value.

  • Assume (to begin with) that the user will enter an integer greater than 0.
  • Note that if you divide an integer by 10 (the base of our number system) the answer will have one less digit than the original. If you divide an integer with only one digit by 10 the answer is 0. For any positive integer, the number of times you can divide by 10 before reaching 0 is equal to the number of digits.
  • Work out the algorithm for a loop which will solve this problem.

    Remember when you are planning an algorithm, it is not necessary to work these things out in the order given here, think about each of them in any order which seems logical to you (this will probably be different for different problems).

    Note: The “solutions” given below are one way of solving the problem; others would be possible (some of them worse, some quite possibly better). Generally it is a good idea to spend a bit of time thinking of alternative ways of solving the problem.

    Consider:

    • Initialisation: what needs to be done before the loop begins

      Declare an int variable to hold the number of digits and initialise it to a suitable value. Think about the correct value to initialise it to. For instance,

      • zero: seems to make sense, as haven’t counted any digits yet
      • one: this also makes sense since all numbers (even zero) are considered (for this problem) to have at least one digit;
      Note: The program already has code to declare a variable to hold the number entered by the user, and code to prompt for their input.
    • Test:
      • What? Remember, the loop body executes again if the test evaluates to true.

        Loop continues while number has not yet reached zero. Possible tests include:

        • number > 0
        • number != 0
      • Where? Before entering the loop or after?

        Consider both

        • testing before. This would work, if the number is zero the loop will not execute at all (OK because our initial implementation does not need to cope with 0).
        • testing after. This could work too, if the number is zero the loop body will execute once, this will cause the number of digits to increment which will be OK if we initialise the number of digits to 0 at the start. Our program would then cope with zero.
    • Body: what needs to be done over and over
      • divide number by 10 (and store the result back into number variable)
      • increment the number of digits by 1
    • Update: what change will you need to make each time around the loop so that the test will eventually become false? Need to change the value of number (so the test will eventually become false). Dividing number by 10 and storing the answer back in number variable (as done in the body) will achieve this

Decide which loop construct will be best to use. Let’s go with testing before. This means that a while loop is the obvious choice

Write the code to implement this loop.

Here is an upgraded main method which implements the algorithm (as discussed above)

public static void main(String[] args) {
    int number;
    int numDigits;

    System.out.print("Please enter a positive integer: ");
    number = sc.nextInt();
    numDigits = 0;
    System.out.println("The number you entered was: " + number);
    while (number != 0) {
        number = number / 10;
        numDigits = numDigits + 1;
    }
    System.out.println("The number of digits is: " + numDigits);
}

Test your program. Write down the integers you will test it with.

  • Try a single digit
  • Try larger numbers of digits (remember though if you go above the largest value that can be stored in an int the program will crash as nextInt() will be unable to parse (interpret) the entered value as an int
  • For curiosity try:
    • negative numbers (this solution handles them OK)
    • zero (this solution gets it wrong [OK because we were not required to handle 0 at this stage])

An Upgrade

What happens if the user enters the integer 0 (which has 1 digit)? Message says number of digits is 0

Work out how you could modify or enhance your code to take care of this special case.

There are several approaches. We could test separately for 0, for example, by adding before the loop (NOT inside the loop) something like:

if (number == 0) {
    numDigits = 1;
}

In this case the loop could go in the else clause.

A final version of main() using this solution:

public static void main(String[] args) {
    int number;
    int numDigits;

    System.out.print("Please enter a positive integer: ");
    number = sc.nextInt();
    if (number == 0) {
        numDigits = 1;
    } else {
        numDigits = 0;
        System.out.println("The number you entered was: " + number);
        while (number != 0) {
            number = number / 10;
            numDigits = numDigits + 1;
        }
    }
    System.out.println("The number of digits is: " + numDigits);
}

There is another way of thinking about it:

  • We could initialise numDigits to 1 (this makes sense since all numbers must have at least one digit).
  • This would then give the correct answer for 0 but would be wrong for other one digit numbers (since the loop would execute once and increment numDigits to 2).
  • This could be overcome by changing the test so that the loop was executed only when the number had greater than 1 digit, as in number / 10 != 0

2. Desk check a program that implements a loop algorithm

An important skill for programmers is the ability to predict the effect of a program by inspection of the code. This is sometimes called “desk checking”. Can you think of ways in which this would be useful?

Open the file PalindromeTester.java.

Compile and run the program to check that it does what it is supposed to do. (A palindrome is a word which reads the same forwards and backwards eg the words radar and madam are palindromes, the words radio and mister are not.)

The lines of the code are numbered (for convenience) and each line of the table shows the effect of execution of one line of code. The columns of the table show the effect of execution, an explanation of the effect and new values of any variables that have been changed. Note that only the values of variables that are important to the current task are included in the table (i.e., the Scanner object is not mentioned).

public static void main(String[] args) {
    Scanner sc = new Scanner(System.in);
    String word;
    int leftPos, rightPos;
    char leftLet, rightLet;

    System.out.print("Enter a word: ");    
    word = sc.next(); 

    //Initialisation: set initial positions of left and right; find first and last letters
    leftPos = 0;
    rightPos = word.length() - 1;
    leftLet = word.charAt(leftPos);
    rightLet = word.charAt(rightPos);

    //Test before loop (first time compares first and last letters in word)
    while ( (leftLet == rightLet) && (leftPos < rightPos) ) {
        //body and update
        leftPos = leftPos + 1; 
        rightPos = rightPos - 1; 
        leftLet = word.charAt(leftPos); 
        rightLet = word.charAt(rightPos); 
    }

    //if we reached half way then must be a palindrome
    if (leftPos >= rightPos) {
        System.out.println("IS a palindrome");
    } else {
        System.out.println("NOT a palindrome");
    }
}
