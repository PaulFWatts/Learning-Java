Overview

These activities give you practise in reading and writing code that makes use of code written by others. You will write code to instantiate objects of classes written by someone else and then call those objects’ methods (the actions they can perform).

Objectives

After completing these activities you should be able to plan and implement algorithms that use:

  1. objects of prewritten classes
  2. class data and class methods

Preparation

  1. Review the relevant notes and lecture materials on MyLO.
  2. Download the starter files (zipped folder) for these activities. Different files are used in different activities, but it is faster to download the full set all at once. Make sure you unzip the files to your work area.
  3. Check that you can remember how to read and make sense of a method header. The vital components are:
    public Must be there, otherwise you can’t use the method outside the class in which it’s declared
    static Might or might not be there; indicates a class method
    return_type Data that is passed out of the method. Possible values are:
    • void — nothing returned
    • a primitive type (e.g., int, double, char, boolean) — a value of that type is returned
    • a class name (e.g., String) — a reference to an object of that class is returned
    Remember that it is the responsibility of the client (that is you if you are writing a program that uses prewritten classes) to store or use the return value. It is not a compiler error to call a method and ignore its return value
    identifier This is the name of the method; it tells you what to write for the call. Remember the syntax of a call:
    • For an object method: objectIdentifier.methodIdentifier(arguments);
    • For a class method: ClassIdentifier.methodIdentifier(parameters);
    (parameter list) Data that must be passed into the method when it is called. Remember, when calling a method:
    • If there are no arguments required then the parameter list is empty ()
    • If the parameter list is not empty then the arguments must be of a compatible type to those specified in each position of the parameter list

Activities

  1. Create and use objects of prewritten classes
  2. Create an interactive program that uses prewritten classes
  3. Use class data and class methods
