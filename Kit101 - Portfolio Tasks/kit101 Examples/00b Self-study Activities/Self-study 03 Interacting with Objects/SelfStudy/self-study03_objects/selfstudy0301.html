Self-study 3 Activity 1: Create and use objects of prewritten classes

[ ↔ Expand↶ Close all sections ] Only visible sections will be included when printing this document.

Expected outcomes

  • Understand the distinction between classes and objects
  • Know how to recognise and write code that:
    • declares a reference variable for an object
    • instantiates an object
  • Know how to read a method header and use the information in it to call methods “belonging” to objects of prewritten classes including
    • passing parameters correctly
    • making appropriate use of return values.

Starting point

If you haven’t already, download and unzip the Self-study 3 starter files. You will be using StringExample.java, kit101/Balloon.java and AbstractionExample.java in this activity.

Run the program several times (but each time enter altitude 15 and duration 5 for b1. Note that b2 will always have and altitude 20, duration 5. Does the output always look exactly the same?

No. The messages from the two balloons, while they are cruising, seem to be sort of jumbled, as if the two balloons are operating independently. (See the output from two sample runs below.)

In fact, they are operating independently. The Balloon class has been implemented so that Balloon objects can make use of a programming feature known as “multi-threading” where each object can have its code execute (sort of) independently from other objects.

Note: Threads are outside the scope of this unit; you will not be expected to write code to implement this, but notice that you can use objects of the Balloon class without needing to understand how they do this.

Questions/ Further Activities

Good code should be “robust”, this means that it does not easily “break” or fail when the user makes a silly mistake. Try to see how robust the Balloon class is (the answer is not very robust at all, but try to see this for yourself.

  • Try giving nonsensical values for height and duration (e.g., ask it to cruise at an altitude of -50, or for a duration of -20).
  • Try doing things in a nonsensical order, such as calling the cruise method before an altitude or duration is set.
Note: It is the responsibility of the programmer who writes a class to make it robust; this does not happen automatically.
